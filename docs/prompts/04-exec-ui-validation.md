OMNIA — VISUAL VALIDATION MODE

You are not only implementing Phase 02.

You are also responsible for validating visual quality.

Do not consider implementation complete simply because the application compiles.

Visual quality is part of the deliverable.

⸻

Validation Philosophy

The objective is to create a UI that feels:

- Clean
- Calm
- Modern
- Mobile-native

The objective is NOT merely to satisfy TypeScript and build requirements.

Every UI change must be visually validated.

⸻

Required Validation Loop

For every significant UI milestone:

1. Build
2. Run
3. Capture screenshots
4. Analyze screenshots
5. Compare against design goals
6. Identify issues
7. Fix issues
8. Repeat

Do not stop at successful compilation.

⸻

Storybook Validation

For every Storybook component:

Generate screenshots for:

- Light Theme
- Dark Theme
- Interaction States

Review:

- Alignment
- Padding
- Typography
- Contrast
- Spacing
- Visual consistency

Document findings.

⸻

Application Validation

Run the application locally.

Capture screenshots of:

- Home Screen
- Chat Screen Mock
- Drawer
- Settings Screen
- Provider Screen

Evaluate:

- Layout balance
- Whitespace usage
- Typography hierarchy
- Component consistency
- Navigation clarity

⸻

Design Evaluation Criteria

Compare the resulting UI against:

1. ChatGPT Mobile
2. llama.cpp UI

The Omnia UI should feel:

- Simpler
- Cleaner
- Less visually dense

than Open WebUI.

⸻

Visual Defect Checklist

Check for:

- Misaligned elements
- Inconsistent spacing
- Crowded layouts
- Excessive borders
- Excessive colors
- Tiny touch targets
- Poor dark theme contrast
- Typography inconsistencies

Fix before proceeding.

⸻

Screenshot Deliverables

Create:

docs/design/reviews/

phase-02-review-01.md

phase-02-review-02.md

…

For each review include:

Screenshots

Attached screenshots

Findings

Issues identified

Improvements

Changes applied

Outcome

Pass / Fail

⸻

Self-Critique Requirement

After each screenshot review:

Act as a senior product designer.

Critique your own implementation.

Identify:

- Weaknesses
- Design inconsistencies
- Areas that do not match Omnia’s vision

Do not assume the implementation is correct.

⸻

Completion Criteria

Phase 02 is not complete when:

- The app builds

Phase 02 is complete when:

- The app builds
- Storybook works
- Screenshots exist
- Visual reviews exist
- The UI matches Omnia’s design vision

Visual quality is a deliverable.
